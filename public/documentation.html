📊 Vue d'ensemble

Qu'est-ce que l'API du Port de Plaisance Russell ?

L'API du Port de Plaisance Russell est une interface RESTful privée qui permet aux administrateurs de gérer les catways (emplacements d'amarrage) et les réservations du port. Cette API facilite la gestion quotidienne de la capitainerie en offrant des fonctionnalités complètes de création, consultation, modification et suppression des données.

Base URL

http://localhost:3000/api

Authentification

Toutes les requêtes doivent inclure un token JWT dans le header Authorization : 

Authorization: Bearer VOTRE_TOKEN_JWT

Routes principales

Catways

GET /catways - Lister tous les catways
GET /catways/:id - Récupérer un catway
POST /catways - Créer un catway
PUT /catways/:id - Modifier un catway
DELETE /catways/:id - Supprimer un catway

Réservations

GET /catways/:catwayNumber/reservations - Lister les réservations d'un catway
POST /catways/:catwayNumber/reservations - Créer une réservation
DELETE /reservations/:id - Supprimer une réservation

🎓 Tutoriel

Étape 1 : Obtenir un token d'authentification

Avant d'utiliser l'API, vous devez vous connecter pour obtenir un token JWT :

POST /api/login
  Content-Type: application/json

  {
    "email": "votre@email.com",
    "password": "votre_mot_de_passe"
  }

Étape 2 : Lister tous les catways

Une fois authentifié, vous pouvez lister tous les catways disponibles :

GET /api/catways
  Authorization: Bearer VOTRE_TOKEN_JWT

Étape 3 : Créer un nouveau catway

Pour ajouter un catway, envoyez une requête POST avec les données requises :

POST /api/catways
  Authorization: Bearer VOTRE_TOKEN_JWT
  Content-Type: application/json

  {
    "catwayNumber": 10,
    "type": "long",
    "catwayState": "Disponible"
  }

Étape 4 : Créer une réservation

Pour réserver un catway, utilisez son numéro :

POST /api/catways/1/reservations
  Authorization: Bearer VOTRE_TOKEN_JWT
  Content-Type: application/json

  {
    "clientName": "Jean Dupont",
    "boatName": "Mon Bateau",
    "checkIn": "2025-06-15T10:00:00Z",
    "checkOut": "2025-06-20T10:00:00Z"
  }

📖 Glossaire

Définitions

Catway
Emplacement d'amarrage pour un bateau, aussi appelé ponton ou place de port. Chaque catway est identifié par un numéro unique.
JWT (JSON Web Token)
Token d'authentification sécurisé utilisé pour vérifier l'identité de l'utilisateur à chaque requête. Le token est généré lors de la connexion et doit être inclus dans le header Authorization.
Réservation
Enregistrement d'une occupation temporaire d'un catway par un client. Une réservation inclut le nom du client, le nom du bateau, et les dates d'arrivée et de départ.
REST (Representational State Transfer)
Style d'architecture pour les systèmes distribués. L'API respecte les principes REST en utilisant les méthodes HTTP standards (GET, POST, PUT, DELETE) pour effectuer des opérations sur les ressources.
MongoDB
Système de base de données NoSQL orienté documents, utilisé pour stocker toutes les données de l'application (catways, réservations, utilisateurs).
Middleware
Fonction intermédiaire qui traite les requêtes HTTP avant qu'elles n'atteignent les routes finales. Dans cette API, le middleware d'authentification vérifie la validité du token JWT.
Type de catway
Catégorisation du catway selon sa taille : long pour les grands bateaux ou short pour les petits bateaux.
État du catway (catwayState)
Description de l'état actuel du catway : "Disponible", "Occupé", "En maintenance", "En bon état", etc.
